## 주요 작업 내용

### 1. 프로젝트 최초 설계 (v1.0.0)

**사용자 요구사항 (원문 요약):**
- KOSPI/KOSDAQ 상위 10개 종목 자동 추적
- Claude Code CLI + Codex CLI 듀얼 AI 토론 → 최적 매수/매도 결정
- Streamlit 웹 대시보드 (포트폴리오 현황, AI 토론 뷰어, 매매 이력)
- Python 단일 스택 (JS/TS 없음)
- 로컬 PC 단독 실행 (외부 서버 불필요)
- 로컬 설치된 AI CLI를 subprocess로 실행 (SDK는 폴백)
- 적응형 토론 루프: 최대 5라운드, 6가지 중단 조건, 무한루프 방지

**구현 완료 파일:**
- `core/models.py`: SQLAlchemy ORM 7개 테이블
- `core/database.py`: SQLite WAL 모드 + context manager 세션
- `config/settings.py`: pydantic-settings 환경변수 로딩
- `data/pykrx_client.py`: KOSPI 상위 10개 종목 OHLCV 수집
- `data/naver_scraper.py`: NAVER Finance 뉴스 스크래핑
- `data/data_manager.py`: 통합 수집·upsert 오케스트레이터
- `indicators/technical.py`: RSI, MACD, BB, EMA, ATR
- `agents/protocol.py`: MarketContext, AgentMessage, ConsensusResult, ExitReason
- `agents/local_cli_runner.py`: subprocess claude/codex CLI + SDK 폴백
- `agents/orchestrator.py`: 적응형 토론 루프 + 6가지 중단 조건 + 가중 합의
- `trading/risk_manager.py`: 포지션 한도·손절·일일 손실 한도
- `trading/paper_broker.py`: DB 기반 모의거래
- `trading/order_executor.py`: paper/live 라우팅
- `trading/portfolio.py`: 포트폴리오 상태 계산·스냅샷 저장
- `scheduler/tasks.py`: 장중 작업 정의 (08:50/09:10/15:40/16:00 KST)
- `scheduler/job_runner.py`: APScheduler CronTrigger 등록
- `dashboard/pages/1_overview.py` ~ `5_settings.py`: Streamlit 5페이지
- `scripts/setup_db.py`, `scripts/backfill_data.py`
- `start.bat`: Windows 원클릭 실행
- `tests/unit/test_agents.py`: 22개 단위 테스트

### 2. 실시간·배치·수기 갱신 시스템 추가 (v1.1.0)

**사용자 요구사항 (원문 요약):**
"프로젝트가 실행된 상태라면, 뉴스현황, 주식의 변화(일봉, 시간봉, 분봉 등)의 값을 배치를 통하여, 
갱신할 수도 있고 또는 수기로도 갱신이 가능하도록 해야해. 즉시 반영이 필요한 경우도 있고, 
이걸 자료로 받아오면 바로 분석을 진행하여 매수 또는 매도에 대한 선택을 할 수 있기 때문이야."

**구현 완료 파일:**
- `data/realtime_collector.py`: DataRefreshManager 싱글턴
  - `refresh_now()`: 수기 즉시 갱신
  - `refresh_intraday_batch()`: 분봉·시간봉 배치 갱신
  - `register_analysis_trigger()`: AI 분석 콜백 등록
  - 인메모리 가격 캐시 + 0.5% 변화 감지 시 즉시 AI 재분석
- `api/routes/refresh.py`: 수기 갱신 REST API
  - `POST /api/refresh/manual`: 비동기 갱신
  - `POST /api/refresh/manual/sync`: 동기 갱신
  - `POST /api/refresh/intraday`: 분봉·시간봉 갱신
  - `GET /api/refresh/prices`: 최신 가격 캐시
  - `GET /api/refresh/status`: 마지막 갱신 시각
- `dashboard/components/refresh_panel.py`: Streamlit 수기 갱신 UI
- `scheduler/job_runner.py`: 장중 5분마다 `job_intraday_refresh` 추가
- `scheduler/tasks.py`: `run_analysis_for_ticker()` 분리 (콜백 등록 가능)

### `_detect_oscillation()` 버그 (agents/orchestrator.py)
- **문제**: 에이전트별로 개별 오실레이션 감지 → [claude=BUY, codex=SELL, claude=BUY] 패턴 미감지
- **원인**: claude 시퀀스 [BUY, BUY]는 교대가 아님 → False 반환
- **수정**: 전체 신호 시퀀스에서 비-HOLD 신호만 추출, 인접 쌍 모두 다를 때만 True 반환
- **검증**: 단위 테스트 `test_oscillation_detection` 통과
