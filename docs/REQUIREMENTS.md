## 1. 프로젝트 배경 및 목적

개인 투자자가 국내 주식(KOSPI/KOSDAQ)을 더 효율적으로 운용하기 위한 AI 보조 자동매매 시스템.
사람이 직접 차트를 분석하고 매매 타이밍을 결정하는 대신, AI가 기술적 분석·뉴스 감정 분석을 수행하고
두 개의 독립 AI(Claude + Codex)가 서로 토론하여 최적 매수/매도 결정을 내린다.

## 4. 요구사항 변경 이력

### v1.0 → v1.1 (2026-06-23)

**변경 1: 프론트엔드 스택 변경**
- **원안**: React + Vite (TypeScript) 대시보드
- **변경**: Streamlit (Pure Python) 대시보드
- **이유**: "Python을 주 스택으로" 요구사항 반영. JS/TS 제거로 단일 언어 스택 유지.

**변경 2: AI 실행 방식 변경**
- **원안**: Anthropic SDK + OpenAI SDK 직접 호출
- **변경**: 로컬 설치된 `claude` CLI + `codex` CLI를 `subprocess`로 실행, SDK는 폴백으로만 사용
- **이유**: "로컬 PC에 설치된 AI를 통해 소통하는 멀티 오케스트레이션" 요구사항 반영

**변경 3: AI 토론 구조 변경**
- **원안**: 고정 3라운드 (Claude → Codex → Claude)
- **변경**: 적응형 최대 5라운드 + 수렴/중단 조건 기반 조기 종료
- **이유**: "서로 가장 최적화된 협의점을 찾아 종료, 무한루프 방지, 중도 중지 룰 필요" 요구사항 반영

### v1.1 → v1.2 (2026-06-24)

**변경 1: 분석 방법론 확장 (1단계)**
- **원안**: 기술지표 16종 + 뉴스 헤드라인만으로 AI 판단
- **변경**: 뉴스 감성분석 점수화(A-09), 퀀트 팩터 지표 모멘텀/이격도 추가(A-10)
- **이유**: "인터넷상의 분석/예측 방법론을 조사해서 적용" 요청 반영. 조사 내역은
  `docs/ANALYSIS_METHODS.md` 참조

**변경 2: 향후 확장 항목 등록**
- DART 펀더멘털(A-11/D-11), 수급 데이터(A-12/D-12)는 중기 과제로,
  ML/DL 예측(F-07), 멀티에이전트 세분화(F-08), RL 전략(F-09)은 장기 백로그로 등록

### v1.2 → v1.3 (2026-06-24)

**변경 1: AI 토론 세션 간 연속성 추가 (A-13)**
- **원안**: `run_debate()` 호출마다 과거 분석 이력을 전혀 참조하지 않고 매번 새로 판단
  (한 토론 세션 *내부*의 라운드 간 히스토리만 있었음)
- **변경**: 동일 종목의 최근 N개(기본 3개) 과거 세션 요약을 모든 `MarketContext` 생성
  지점에서 자동 조회해 모든 라운드 프롬프트에 반영
- **이유**: "AI들이 실행될 때마다 이전 대화 내용도 알고 토론이 이어져야 한다"는 요청 반영.
  규칙 상세는 `docs/AI_AGENT_PROTOCOL.md`의 "세션 간 연속성 정책" 절 참조

### v1.3 → v1.4 (2026-06-24)

**변경 1: 월/주/연봉 데이터 + JSON 아카이브/재적재 (D-14)**
- **원안**: 일봉만 수집·저장
- **변경**: `data/pykrx_client.py`에 월/연봉(pykrx freq) + 주봉(일봉 resample 파생) 수집 함수
  추가, `scripts/collect_price_archive.py`/`load_price_archive.py`로 종목+시간단위 단위 JSON
  백업·재적재. 분봉은 pykrx가 지원하지 않아 사용자 결정으로 이번 범위에서 제외
- **이유**: "1년치 연/월/주/일봉 데이터 수집 서비스 + 종료 시 휘발 대응" 요청 반영.
  상세는 `docs/DATA_COLLECTION_ARCHITECTURE.md` 참조

**변경 2: KRX 휴장일 인식 (D-13)**
- **원안**: `is_market_open()`이 주말만 체크, 한국 공휴일에도 스케줄 작업이 실행됨
- **변경**: `config/market_calendar.py` 정적 캘린더 + `is_trading_day()` 추가, 일 1회짜리
  스케줄 작업 4개(premarket/morning_analysis/news_refresh/eod_snapshot) 모두 휴장일 가드 적용
- **이유**: 가이드라인 작성 중 발견한 간극. `docs/PROJECT_REVIEW_2026-06-24.md` P1-8과 동일 이슈

**변경 3: D-06 상태 정정**
- **원안**: "✅ 완료"로 표시
- **변경**: "⚠️ 오류"로 정정 — pykrx가 분봉을 지원하지 않고 기존 구현이 존재하지 않는 함수
  시그니처를 호출하는 버그 상태임을 직접 코드 확인으로 검증
- **이유**: `docs/PROJECT_REVIEW_2026-06-24.md` P0-3과 동일 결함을 독립적으로 확인. 문서의
  완료 표시는 실제 코드로 교차 확인해야 한다는 `AGENTS.md` 원칙 반영
